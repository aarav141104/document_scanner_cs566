Robust Automatic Document Scanner Using Classical Computer Vision

Author: Aarav Anand
Course: CS 566 (Fall 2025)

1. Motivation

Photos of documents taken on phones are often skewed, shadowed, and difficult to read. The goal of this project is to automatically detect the document region in a natural image and correct the perspective so that it appears as a clean, top-down scanned page.

2. Approach

My approach uses a classical computer vision pipeline:

  • Gaussian blur to reduce noise
  • Adaptive thresholding to handle uneven lighting
  • Morphological closing and dilation to strengthen weak edges
  • Contour detection with convex-hull fallback for robust document boundary detection
  • Corner ordering (TL, TR, BR, BL)
  • Homography-based perspective transform to flatten the document

3. Implementation

The implementation was done in Python using OpenCV. Below is a high-level overview of the main steps: 

1. Read and preprocess the image  
2. Apply adaptive thresholding  
3. Use morphology to clean and enhance edges  
4. Detect largest document-like contour  
5. Order points  
6. Compute homography and warp the image

5. Problems Encountered & What I Learned

  • Simple Canny edge detection failed on noisy backgrounds
  • Shadows and faint document boundaries required stronger preprocessing

6. Interesting Observations / Comparisons

Adaptive thresholding plus morphology performed far better than simple edge detection. Convex hull fallback significantly improved robustness on weak-edge images.
